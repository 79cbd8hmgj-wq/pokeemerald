EMERALD_BREEDING_TASK_MATRIX_v1
Compact build matrix for current breeding rework plan
Master authority: pokemon_gen1_3_release_polished_updated_types_v11.txt

==================================================
PURPOSE
==================================================

This file converts the audited breeding plan into a compact implementation matrix.
It is meant to help with actual build order, scripting scope, and balance testing.

Included statuses:
- KEEP
- REWRITE
- LIGHTEN

Excluded from main matrix:
- CUT species

Core design rules:
- v11 identity always leads
- breeding should refine role, not replace it
- no flat BST rewards for bred Pokemon
- hatch rewards should be curated, readable, and limited

==================================================
FIELD DEFINITIONS
==================================================

Status
- KEEP = current breeding concept still fits v11 well
- REWRITE = species needs a rebuilt breeding plan around current v11 identity
- LIGHTEN = breeding support kept, but reduced because v11 already did major work

Breeding Role
- short statement of what breeding is trying to accomplish

Trigger Type
- the kind of breeding condition used to activate a reward

Reward Type
- what the offspring actually gets

Risk
- Low / Medium / High

Implementation Difficulty
- Easy / Medium / Hard

Phase Order
- Phase 1 = first implementation batch
- Phase 2 = second implementation batch
- Phase 3 = later stable additions
- Phase 4 = low-intensity finishing additions
